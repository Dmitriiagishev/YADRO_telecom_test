# YADRO_telecom_test

## Бот данжн кроулер
Алгоритм Алисы можно улучшить множеством способов.
### Самый простой способ
На обратном пути бота, собирать не все ресурсы, начиная с первой комнаты, а, запомнить, в каких комнатах, по пути до данной, были какие ресурсы, после чего, собирать только лучшие из возможных вариантов.
То есть, при возвращении, осталось пройти n комнат. У игрока m лишней еды. в каждой комнате осталось по 3 возможных варианта подбора ресурсов (т.к. из 4 возможных, при первом проходе, бот уже собрал лучшие.) Из всех оставшихся 3n вариантов выбираются m лучших, после чего, работо собирает только их в соответствующих комнатах.
(Минусов нет)
### Улучшенная карта
При первом исследовании подземелья, робот может вести учет тех комнат, которые он прошел и их соседей. если у двух пройденных комнат есть одинаковый сосед, которого не посетили - его можно использовать, как возможность альтернативного пути. 
Например, есть комнаты: 1 - соседничает с 2, 3; 4 - также, соседняя с 2, 3 (из 2 в 3 прохода нет). при первом исследовании, робот поедет по маршруту 1 -> 2 -> 4 и соберет оттуда лучшие ресурсы без траты еды. на обратном пути он поедет тем же маршрутом в обратную сторону, хотя, мог бы, поехать 4 -> 3 -> 1, и собрать бесплатно лучший ресурс в 3.
(Может произойти ситуация, в которой 2 лучших ресурса вместе в новой комнате будут хуже лучшего из оставшихся. В среднем, дополнительный бесплатный ресурс почти всега будет лучше)
### Чем глубже в лес - тем больше дров
Вместо того, чтобы исследовать подземелье, пока не останется фиксированно половина запаса еды, можно, после каждого перехода рассчитывать, какова минимальная длина пути до начальной комнаты. Если, после очередного перехода еды не хватит - нужно возвращаться кратчайшим путем и собрать лучший ресурс по дороге (если есть возможность). Если же, после перехода, еды ещё хватит, чтобы вернуться - можно перейти и бесплатно получить еще ценности.
(Может возникнуть ситуация, аналогичная прошлому пункту)
